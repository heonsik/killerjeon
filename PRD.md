# 킬러전 Roblox 게임 PRD

## 1. 개요

`킬러전`은 Roblox에서 여러 명의 플레이어가 넓은 방 맵 안에서 추격자를 피해 살아남는 생존형 라운드 게임이다. 플레이어는 제한 시간 동안 숨거나 도망치고, 맵에 있는 아이템을 활용해 생존 확률을 높인다.

초기 버전은 자녀와 함께 완성 가능한 범위로 줄여서, 복잡한 전투보다 `도망`, `숨기`, `시간 버티기`, `간단한 아이템 사용`에 집중한다.

## 2. 목표

- Roblox Studio에서 실제로 플레이 가능한 1차 버전을 만든다.
- 한 라운드가 시작되고 끝나는 기본 게임 흐름을 구현한다.
- 플레이어가 추격자에게 잡히지 않고 제한 시간 동안 버티는 재미를 만든다.
- 이후 무기, 스킨, 추가 맵, 보스 몬스터 등을 확장할 수 있는 구조로 만든다.

## 3. 대상 플레이어

- Roblox를 좋아하는 어린이 및 가족
- 친구와 함께 짧게 즐길 수 있는 생존 게임을 원하는 플레이어
- 어렵지 않은 조작으로 긴장감 있는 술래잡기 게임을 즐기고 싶은 플레이어

## 4. 게임 콘셉트

### 게임 이름

- 임시 이름: `킬러전`
- 추천 순화 이름 후보:
  - `추격자 방탈출`
  - `도망쳐! 큰 방`
  - `전의 생존방`

### 장르

- 숨바꼭질/생존
- 배틀/격투 요소 일부
- 아이템 수집

### 배경

- 넓은 방
- 책상, 의자, 침대, 상자, 장난감, 옷장 같은 큰 물건들이 있는 실내 맵
- 플레이어는 작은 캐릭터처럼 방 안을 돌아다닌다.

## 5. 핵심 게임 루프

1. 플레이어가 로비에 입장한다.
2. 일정 인원이 모이거나 시작 버튼을 누르면 라운드가 시작된다.
3. 플레이어들이 넓은 방 맵으로 이동한다.
4. 10초 준비 시간이 지난 뒤 추격자가 등장한다.
5. 플레이어는 제한 시간 동안 도망치거나 숨는다.
6. 추격자에게 잡히면 탈락하거나 관전 상태가 된다.
7. 시간이 끝날 때 한 명 이상 살아 있으면 생존자 승리.
8. 제한 시간 안에 모든 플레이어가 잡히면 추격자 승리.
9. 결과 화면 후 다시 로비로 돌아간다.

## 6. 1차 MVP 범위

1차 버전에서는 반드시 필요한 기능만 구현한다.

### 포함 기능

- 로비 공간
- 게임 시작 버튼
- 넓은 방 맵 1개
- 라운드 타이머
- 준비 시간
- NPC 추격자 1명
- 추격자 접촉 판정
- 플레이어 체력 또는 탈락 처리
- 생존자 승리/추격자 승리 판정
- 결과 메시지
- 기본 아이템 2종

### 제외 기능

아래 기능은 1차 버전 이후 업데이트로 미룬다.

- 총, 바주카, 전기톱 등 복잡한 무기
- 여러 종류의 몬스터
- 캐릭터 스킨 상점
- 점수 랭킹
- 여러 맵
- 퀘스트 시스템
- 보스전
- 모바일 전용 UI 최적화

## 7. 게임 규칙

### 플레이 인원

- 최소 1명
- 최대 10명

### 라운드 시간

- 준비 시간: 10초
- 생존 시간: 180초

### 승리 조건

- 생존자 승리: 시간이 끝날 때 한 명 이상 살아남는다.
- 추격자 승리: 제한 시간 안에 모든 생존자가 잡힌다.

### 탈락 조건

- 추격자에게 닿으면 체력이 감소한다.
- 1차 버전에서는 단순화를 위해 추격자에게 1번 잡히면 탈락으로 처리한다.
- 추후 업데이트에서 체력 3칸 방식으로 확장할 수 있다.

## 8. 캐릭터와 역할

### 생존자

- 모든 플레이어의 기본 역할
- 목표는 제한 시간 동안 살아남는 것
- 맵을 돌아다니며 숨거나 아이템을 사용한다.

### 추격자

- 1차 버전에서는 NPC로 구현한다.
- 가장 가까운 생존자를 따라간다.
- 생존자와 닿으면 해당 생존자를 탈락시킨다.

## 9. 아이템

1차 버전 아이템은 구현 난이도를 낮추기 위해 단순한 효과만 사용한다.

### 회복약

- 추후 체력 시스템이 들어갈 때 사용한다.
- 1차 MVP에서는 선택 구현으로 둔다.

### 속도 포션

- 사용하면 5초 동안 이동 속도가 빨라진다.
- 도망치는 재미를 만들기 쉽기 때문에 1차 버전 우선 아이템으로 사용한다.

### 열쇠

- 맵에 3개 배치한다.
- 1차 MVP에서는 선택 기능이다.
- 확장 버전에서는 열쇠를 모두 모으면 탈출문을 열 수 있다.

## 10. 맵 요구사항

### 기본 맵

- 넓은 방 하나
- 플레이어가 숨을 수 있는 장애물 6개 이상
- 이동 가능한 길이 막히지 않도록 구성
- 추격자가 너무 쉽게 잡지 못하도록 우회로 배치

### 추천 오브젝트

- 책상
- 침대
- 의자
- 장난감 상자
- 옷장
- 책장
- 커튼 뒤 숨는 공간

## 11. UI 요구사항

### 로비 UI

- 게임 시작 버튼
- 현재 상태 표시: 대기 중, 준비 중, 게임 중

### 게임 중 UI

- 남은 시간
- 살아남은 플레이어 수
- 내 상태: 생존 또는 탈락

### 결과 UI

- 생존자 승리 메시지
- 추격자 승리 메시지
- 로비로 돌아가는 안내

## 12. 기술 요구사항

### Roblox 서비스

- `ReplicatedStorage`: RemoteEvent, 공통 설정값 저장
- `ServerScriptService`: 라운드 진행, 승리 판정, NPC 제어
- `StarterGui`: 타이머와 상태 UI
- `Workspace`: 로비, 맵, 스폰 위치, 아이템 배치

### 주요 스크립트 후보

- `RoundManager`
  - 라운드 상태 관리
  - 준비 시간/게임 시간 관리
  - 승리 조건 확인

- `ChaserController`
  - NPC 추격자 이동
  - 가장 가까운 플레이어 추적
  - 접촉 시 탈락 처리

- `ItemManager`
  - 아이템 생성
  - 아이템 획득 처리
  - 속도 포션 효과 적용

- `GameUI`
  - 타이머 표시
  - 생존자 수 표시
  - 결과 메시지 표시

## 14. 개발 순서

### 1단계: 기본 맵

- 로비 만들기
- 넓은 방 맵 만들기
- 플레이어 스폰 위치 만들기
- 장애물과 숨을 공간 배치하기

### 2단계: 라운드 흐름

- 시작 버튼 만들기
- 준비 시간 구현
- 게임 시간 타이머 구현
- 시간이 끝나면 결과 표시

### 3단계: 추격자

- NPC 추격자 만들기
- 가까운 플레이어를 따라오게 만들기
- 닿으면 플레이어 탈락 처리

### 4단계: 승리/패배

- 생존자 수 계산
- 모두 잡히면 추격자 승리
- 시간이 끝나면 생존자 승리
- 라운드 종료 후 로비로 이동

### 5단계: 아이템

- 속도 포션 추가
- 아이템 획득 처리
- 사용 후 일정 시간 뒤 효과 해제

## 15. 2차 업데이트 후보

- 플레이어 중 한 명이 추격자가 되는 모드
- 열쇠 3개를 모아 탈출하는 모드
- 여러 종류의 추격자
- 무기와 방어 아이템
- 캐릭터 스킨
- 코인 보상
- 맵 투표
- 난이도 선택
- 보스 몬스터 등장

## 16. 성공 기준

1차 버전은 아래 조건을 만족하면 완성으로 본다.

- 플레이어가 로비에서 게임을 시작할 수 있다.
- 라운드가 시작되면 방 맵으로 이동한다.
- 타이머가 정상적으로 줄어든다.
- 추격자가 플레이어를 따라온다.
- 잡힌 플레이어는 탈락한다.
- 시간이 끝나거나 모두 잡히면 결과가 나온다.
- 다시 로비로 돌아갈 수 있다.
